TypeScript Tutorial for JavaScript Programmers
 TypeScript is a programming language built strongly off the idea of typing, as you might have guessed. When compiled, typescript files are turned into JavaScript files that can be understood by the browser, which makes this language useful for web development. TypeScript is also able to communicate directly with your HTML files in order to perform actions on your webpage. As a JavaScript developer, you can think of TypeScript as a more strongly typed JavaScript which can perform the same actions as JS, and then some.


Below is a code example showing a bit of what you can do with TypeScript.
function turnRed() {
  if(document != null){
   let c = document.getElementById("demo").style.color;
  if(c === 'red'){
   document.getElementById("demo").style.color = "black";
  }else{
   document.getElementById("demo").style.color = "red";
   }
  }
 }

Types

TypeScript is essentially an extension of JavaScript, and thus it knows the language and the types. The set of primitive types are as follows: boolean, bigint, null, number, string, symbol, and undefined.

Type by Inference

One way typing works in TS is by inference, where it looks at the value that was assigned to something and determines what it is.
let name = "John Doe";
let age = 24;

In the examples above, TypeScript looks at the values John Doe and 24, and identifies that they are a string and number respectively. This way of typing helps reduce clutter, because you don't need code to explicitly say what the type is.

Defining Types

Inference typing may not always work, such as for creation of objects. To help, TypeScript supports the extension that allows for you to tell TS what the types should be. One way to do this is by creating an object with inferred typing, or by using an interface as shown below.
interface student {
  name: string;
  grade: number;
}

const student: Student = {
  name: "John",
  grade: 11,
};

Structural Type System

In inference typing TypeScript looks at the value assigned, but it can also look at the structure of an object to determine type. In this system, if two objects have the same shape, they are inferred to be the same type. For example, if an interface us created but the object isn't created by means of the interface. TypeScript looks at the shape of these objects to determine similarity/type.
interface Bank {
  id: string;
  accountNumber: number;
}

const bank = { id: "johnbank123", accountNumber: 123456789 };

In the example above, the code to instantiate the 'bank' variable did not use the interface syntax in the previous example, but since the shape of the interface is the same as the object, TypeScript would infer that 'bank' is a Bank object.
